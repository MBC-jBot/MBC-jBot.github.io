# 馬達與編碼器

MBC-jBot 具備強大的雙輪驅動系統，並且在輪軸上搭載了高精度的硬體編碼器（Encoder），讓小車不僅能跑，還能「知道自己跑了多遠」。

## 控制馬達移動

您可以獨立控制左右輪，或者同時控制雙輪。馬達的速度範圍介於 **-100 到 100** 之間。
*   正數 (`1` ~ `100`)：前進
*   負數 (`-1` ~ `-100`)：後退
*   零 (`0`)：停止（滑行）
*   煞車 (`127` 或使用常數 `BRAKE`)：緊急煞車鎖死馬達

### 1. 同時控制雙輪 (`setMotors`)

這是最常用的移動指令。

```cpp
// 語法：Bot.setMotors(左輪速度, 右輪速度);

Bot.setMotors(50, 50);   // 以 50% 速度直線前進
Bot.setMotors(-50, -50); // 以 50% 速度直線後退
Bot.setMotors(50, -50);  // 原地右轉 (左輪往前，右輪往後)
Bot.setMotors(0, 0);     // 停止並滑行
Bot.setMotors(BRAKE, BRAKE); // 緊急煞車
```

### 2. 獨立控制單輪 (`setMotor`)

如果您只需要控制其中一個輪子，可以使用這個指令。
參數 `index` 必須填寫常數 `MOTOR_L` (左輪) 或 `MOTOR_R` (右輪)。

```cpp
// 語法：Bot.setMotor(馬達編號, 速度);

Bot.setMotor(MOTOR_L, 80); // 只有左輪以 80% 速度前進
Bot.setMotor(MOTOR_R, BRAKE); // 右輪緊急煞車
```

> [!NOTE]
> **為什麼是 127 煞車？**
> 在底層程式碼中，`int8_t` 的範圍是 -128 到 127。我們巧妙地利用最大值 `127` 來觸發馬達驅動晶片的「短路煞車」模式，這比單純輸入 `0` 滑行能更快讓車子停下。

## 讀取編碼器 (Encoder)

編碼器就像是小車的「計步器」，它會計算輪子轉動了幾度。MBC-jBot 的底層使用了非常高效的「硬體中斷 (PCINT2)」與「原子操作 (ATOMIC_BLOCK)」技術來確保計步不漏接。

### 1. 取得行駛步數 (`getEncoder`)

您可以讀取左輪、右輪，或是兩輪的平均值。
參數 `index` 必須填寫：
*   `ENC_L`: 取得左輪步數
*   `ENC_R`: 取得右輪步數
*   `ENC_AVG`: 取得左右輪步數的平均值 (適合用來計算直線前進的總距離)

```cpp
// 語法：int32_t Bot.getEncoder(編碼器編號);

int32_t leftTicks = Bot.getEncoder(ENC_L);
int32_t avgTicks = Bot.getEncoder(ENC_AVG);

Serial.print("目前平均行駛步數：");
Serial.println(avgTicks);
```

> [!TIP]
> **如何將步數 (Ticks) 轉換為實際行駛距離？**
> 根據原廠規格，小車的越野車輪直徑約為 **34 mm**，而馬達轉動完整一圈會產生約 **640 個步數**。
> 
> *   車輪周長 = 直徑 × 圓周率 = 34 × 3.14159 ≈ **106.8 mm (約 10.68 公分)**
> *   每走 1 公分需要的步數 = 640 ÷ 10.68 ≈ **60 步**
> 
> 因此，如果您希望小車精確**前進 10 公分**，只要寫一個 `while` 迴圈讓馬達轉動，直到 `Bot.getEncoder(ENC_AVG)` 的數值大於 `600` 時煞車即可！

### 2. 重置編碼器歸零 (`resetEncoders`)

當您準備進行下一段精確移動時，通常會需要先把計步器歸零。
參數 `index` 必須填寫 `ENC_L`、`ENC_R` 或 `ENC_ALL` (同時歸零雙輪)。

```cpp
// 語法：Bot.resetEncoders(編碼器編號);

Bot.resetEncoders(ENC_ALL); // 將左右輪的計步器同時歸零
```
