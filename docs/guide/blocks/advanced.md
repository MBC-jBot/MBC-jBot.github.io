# 5. 進階：序列埠與編碼器

如果您已經熟悉了基礎控制，接下來可以進入工程師領域，學習如何使用「編碼器」讓小車精確地走位，或是用「序列埠」來回傳感測器的偵測數據！

## 積木總覽

*   **重置馬達編碼器積木**：把車輪的「計步器」歸零。
*   **讀取編碼器數值積木**：讀取左輪、右輪或雙輪平均累積轉動的步數 (Ticks)。
*   **讀取馬達速度積木**：讀取馬達當前的運行速度。
*   **序列埠列印積木**：把文字或數字透過 Type-C 線傳回電腦，顯示在序列埠監控視窗中。

### 💡 教學小提示

*   **步數怎麼算？**：根據原廠規格，小車的越野車輪直徑約為 34 mm，馬達轉動完整一圈會產生約 640 個步數。換算下來，**每前進 1 公分大約需要 60 步**。
*   **迴圈的妙用**：我們利用了「重複直到 (whileUntil)」迴圈。只要平均步數還「小於」 600，小車就會不斷執行迴圈裡面的「前進」指令。一旦步數到達 600，它就會跳出迴圈，執行下一步的「煞車」！
*   **序列埠即時監控**：在迴圈中，我們使用「透過序列埠列印」積木將平均編碼器數值即時回傳給電腦。只要打開軟體中的序列埠監控視窗，您就能像汽車儀表板一樣，看到小車實際走過的步數不斷增加！
